My Homelab: Mini PCs to K3s
From three Beelink EQ12 mini PCs to a fully functional Kubernetes cluster — using Ubuntu Server 24.04, K3s, LUKS full-disk encryption, Prometheus, and Grafana. The kind of thing Factorio players do on weekends.

Motivation

I've always been drawn to automation and interconnected systems. That pull was probably first sparked by Minecraft's automatic farms, later solidified by Factorio — design intricate factories to automate production towards launching a rocket. Incredible dopamine. That love for building complex systems naturally led me to Kubernetes.

Why K3s?

K3s seemed like the right starting point: a solid Kubernetes foundation that doesn't abstract away too much, unlike MicroK8s from Canonical. I wanted the challenge of navigating real complexities to build genuine understanding. Eventually I might migrate to Talos Linux for its immutable, API-driven approach paired with standard K8s.

Hardware

Three Beelink EQ12 mini PCs — low power consumption, sufficient compute, perfect for 24/7 operation.

CPU Intel N100 — 10c/12t per node (30c/36t total) RAM 24 GB DDR5 per node Storage 500 GB NVMe SSD per node Network Gigabit Ethernet switch Nodes honey1 (control), honey2, honey3 (workers)

Operating System

Tried Talos Linux first — quickly out of my depth. Considered Arch, but anticipated maintenance headaches on a server. Landed on Ubuntu Server 24.04.2 LTS: familiar apt, rock-solid, well-documented for Kubernetes workloads.

Added LUKS full-disk encryption on each node. The drawback is manually entering a decryption passphrase at boot. For now: a Bluetooth keyboard that switches between all three machines. Eventually I'll set up a Tang server for automated network-bound unlocking.

Installing K3s

Disable ufw first

BASH█
sudo ufw disable
sudo systemctl disable ufw

Control plane on honey1

BASH█
curl -sfL https://get.k3s.io | sh -
BASH█
sudo cat /var/lib/rancher/k3s/server/node-token

Join honey2 and honey3

BASH█
curl -sfL https://get.k3s.io | \
  K3S_URL=https://:6443 \
  K3S_TOKEN= sh -

kubectl on your workstation

  1. Copy /etc/rancher/k3s/k3s.yaml from honey1
  2. Place at ~/.kube/config
  3. Change 127.0.0.1 to honey1's actual IP
BASH█
kubectl get nodes   # all three nodes Ready

Monitoring: Prometheus + Grafana

Helm install

BASH█
helm repo add prometheus-community \
  https://prometheus-community.github.io/helm-charts
helm repo update

helm install prometheus-stack \
  prometheus-community/kube-prometheus-stack \
  --namespace=prometheus-stack \
  --create-namespace

Expose Grafana

YAML█
apiVersion: v1
kind: Service
metadata:
  name: grafana-service
  namespace: prometheus-stack
spec:
  type: NodePort
  selector:
    app.kubernetes.io/name: grafana
  ports:
    - port: 3000
      targetPort: 3000
      nodePort: 30001
YAML█
apiVersion: traefik.containo.us/v1alpha1
kind: IngressRoute
metadata:
  name: grafana-ingressroute
  namespace: prometheus-stack
spec:
  entryPoints:
    - web
  routes:
    - match: Host(`grafana.local`)
      kind: Rule
      services:
        - name: grafana-service
          port: 3000
BASH█
kubectl apply -f .
# then open http://grafana.local
