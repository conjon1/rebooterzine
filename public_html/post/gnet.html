Tunneling All System Traffic Through Your Phone with SSH, redsocks, and Mullvad

TESTED ON: Ubuntu 24.04 LTS · Android 14 · Mullvad
$

Overview

This setup routes all TCP traffic from your Linux box through a SOCKS5 tunnel over USB tethering, landing on an Android phone running Mullvad VPN. DNS is encrypted via dnscrypt-proxy over DoH. IPv6 is disabled at runtime to plug leaks.

[WARN] This will break your normal network path. Keep a fallback TTY open.
$

1. Enable USB tethering

On Android, go to Settings → Hotspot & tethering → USB tethering. Your phone will appear as usb0 on the host.

root@reboot:~#ip link show usb0
$

2. Install redsocks

bash
sudo apt install redsocks
$

3. Configure redsocks

Edit /etc/redsocks.conf:

conf
base {
    log_debug = off;
    log_info = on;
    daemon = on;
    redirector = iptables;
}

redsocks {
    local_ip = 127.0.0.1;
    local_port = 12345;
    ip = 127.0.0.1;
    port = 1080;        // SSH SOCKS5 port
    type = socks5;
}
/etc/redsocks.conf
/etc/dnscrypt-proxy/dnscrypt-proxy.toml
/etc/systemd/system/redsocks.service
$

4. Open the SSH SOCKS5 tunnel

root@reboot:~#ssh -D 1080 -N -f user@your-vps
[TIP] Add -o ServerAliveInterval=60 to keep the tunnel alive.
$

5. Redirect traffic with iptables

bash
# Exclude the SSH server itself from redirection
iptables -t nat -A OUTPUT -d YOUR_VPS_IP -j RETURN
# Redirect everything else
iptables -t nat -A OUTPUT -p tcp -j REDIRECT --to-ports 12345
$

6. Disable IPv6

bash
sysctl -w net.ipv6.conf.all.disable_ipv6=1
sysctl -w net.ipv6.conf.default.disable_ipv6=1
[INFO] These settings are not persistent. Add them to /etc/sysctl.conf for permanence.
