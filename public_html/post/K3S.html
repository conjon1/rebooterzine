My Homelab Journey: From Mini PCs to Kubernetes with K3s

From a few mini PCs to a fully functional Kubernetes cluster – my homelab journey using Ubuntu and K3s has been full of learning experiences (and a few headaches!).

$

Motivation: The Pull of Automation

I've always been drawn to automation and building effective, interconnected systems. This interest was likely first sparked by games like Minecraft with its automatic farms, and later solidified by titles like Factorio and Mindustry. In Factorio, in particular, you design intricate, interconnected factories to automate production towards launching a rocket – it's incredible dopamine, like candy for my brain. This love for building complex systems naturally led me to explore the world of container orchestration with Kubernetes.

Why K3s?

When I discovered K3s, I was immediately interested. It seemed like a great starting point: providing a solid Kubernetes foundation that was relatively easy to set up but didn't abstract away too much, unlike options like MicroK8s from Canonical.

$

The Hardware

For my Kubernetes homelab, I chose 3x Beelink EQ12 mini PCs. These compact machines offered the perfect blend of low power consumption and sufficient processing power.

  • Processor: Each unit features an efficient Intel processor.
  • RAM: 24GB DDR5 RAM per unit.
  • Storage: 500GB NVMe SSD per unit.

Across the three nodes, this gives a total of 30 cores and 36 threads, which is a decent amount of processing power for this learning project.

$

The Operating System: Ubuntu with a Security Twist

Initially, I tried running Talos Linux, but I quickly realized I was a bit out of my depth. Getting the system configured exactly as I wanted proved challenging (mostly user error, honestly!). I also considered Arch Linux, but having used it as my primary desktop OS for years, I anticipated potential maintenance headaches down the line.

So, I turned my attention to Debian-based systems, and Ubuntu immediately came to mind. For added security, I opted for LUKS full-disk encryption on each node.

$

Getting Hands-On: Installing K3s on Ubuntu

With the hardware chosen and Ubuntu Server 24.04.2 LTS installed on my three "honey" nodes, it was time to dive into the K3s installation.

Disabling the Firewall

bash
sudo ufw disable
sudo systemctl disable ufw

Installing the K3s Control Plane (on honey1)

bash
curl -sfL https://get.k3s.io | sh -
sudo cat /etc/rancher/k3s/k3s.yaml

Joining the Worker Nodes (honey2 & honey3)

bash
curl -sfL https://get.k3s.io | K3S_URL=https://<honey1-ip>:6443 K3S_TOKEN=<your-token> sh -
$

Monitoring with Prometheus & Grafana

With the cluster running, the next step was setting up a monitoring stack using Prometheus and Grafana via a Helm chart.

bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

helm install prometheus-stack prometheus-community/kube-prometheus-stack \
  --namespace=prometheus-stack \
  --create-namespace

Making Access Easier: NodePort and Ingress

yaml
# grafana-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: grafana-service
  namespace: prometheus-stack
spec:
  type: NodePort
  selector:
    app.kubernetes.io/name: grafana
  ports:
    - protocol: TCP
      port: 3000
      targetPort: 3000
      nodePort: 30001
[TIP] After applying the ingress route, you can easily access the dashboard at grafana.local.
